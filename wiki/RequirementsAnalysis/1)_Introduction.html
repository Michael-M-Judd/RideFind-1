1 Introduction¶

1.1 Purpose of the System¶

As university students, we understand the way you move. RideFind enables you to share your commute with others travelling to the same location, sharing costs and minimizing environmental impact. We aim to connect students with their destinations through safe, reliable, and easy to use service--one ride at a time.

1.2 Scope¶

The major deliverable for this project will be the completed and functional RideFind application. This will involve gathering functional requirements, various pieces of documentation, and both back and front end development. A database will be integrated with the software to hold information about users and rides. A GUI will be developed for the user to interact with the software. This will include pages that allow users to complete functions such as create an account, view open reservations, book a seat in a car, review information about the driver and communicate with other users. The software will also include pages that allow the user to give a user review of the driver after the ride has been completed.

1.3 Core System Functionalities¶

  • Allow up to a theoretically unlimited amount of users to find/create ride listings. Note that there is no ability to save a ride listing in a draft; the users must either post it in the database or exit.
  • Provide a GUI that allows users to scroll through listings and navigate through the database.
  • Allow users to create a profile and view other user profiles.
  • Allow users to select from a number of ride listings.
  • Provide a ride search filter.
  • Allow passengers to rate the driver after the ride is complete.

1.4 Objectives and Success Criteria of the Project¶

  • The design of the GUI to enable intuitive selection of a correct ride based on various preferences (e.g. departure date/time, destination, etc.)
  • The design of a database that allows for the easy and efficient creation and access of ride listings
  • Having a priority queue system so that rides are displayed in order of departure time
  • Powerful search capability
  • Detailed profiles for drivers/passengers so that reviews can be created, improving overall user satisfaction

1.5 Definitions, Acronyms, and Abbreviations¶

  • Back-End Development: The back-end consists of the database storing user information.
  • Completed Ride: When a ride listing has reached its departure date and time and has at least one confirmed passenger.
  • Driver: A user on RideFind who is offering a ride/is looking for passengers. They will create a ride listing and be paid for "sharing their ride" with others.
  • Expiration: When a ride listing has reached its departure date and time and has zero passengers, the system deletes it from the database.
  • Front-End Development: The front-end is an interface between the user and the back-end. The GUI is the main aspect of front end development.
  • GUI: Graphical User Interface, the windows that users interact with.
  • Login Screen: The first page a user sees. They enter their email and password (which is verified) to access the RideFind Main Page.
  • Main Page: The first page a user sees once they have logged into RideFind.
  • Passenger: A user on RideFind who is searching for a ride/looking for a driver. They will look at different ride listings to see what would work best for them and then pay the driver for "sharing their ride".
  • Priority Queue: The ride listings will be displayed in the main page prioritized in terms of spots left and time of departure. A priority queue will be data structure used to implement this.
  • Profile: All users will have created a profile associated with them. Their name, student email, car model, profile picture and summary of recent reviews are included in their profile.
  • Review: Users can provide feedback to indicate their satisfaction or dissatisfaction with a completed ride. They can fill out a number of starts, write a comment or do both.
  • Ride Booking: When a driver and a passenger agree upon the conditions of a ride listing.
  • Ride Listing: Ride listings are displayed on the main page. Details of a ride listing are: driver name, destination, departure location, date of departure, time of departure, seats and cost.
  • Ridesharers: People who believe in the "ridesharing" mantra and use RideFind.
  • Search Bar: Displayed on the main page, a user can change various categories to refine what ride listings are displayed.
  • Spots: This number is based on the number of free spots in a car.
  • Verification: To ensure that a local community of Ridesharers is created, university/college emails are needed to register for RideFind.

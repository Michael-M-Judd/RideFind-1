c.) Persistent Data Management¶

Section Goal: Describe the persistent data stored by the system and the data management infrastructure required for it. This section should include the description of data schemes, the selection of a database, and a description of the encapsulation of the database.

A MySQL database will be used to store information essential to the functioning of the RideFind application. MySQL integrates well with Visual Studio and SQLAPI++ library has been selected for connection and sending commands to the database. The database relation will have four different tables including User, Car, Ride and Ban.

The database will be used to store user information in a User table which contains information about the user such as a unique identifier, name (first and last), birthday, email and role within the application; passenger or driver represented as a binary number.

The database will also store information on the car in the Car table. This relation will store information such as the car's make, model and year and each car will be linked to a driver's identification number.

The ride table in the database will store information on each ride created by a driver. This relation will contain information on departure location, departure time and arrival location. For each user associated with the ride, there will be a new instance of the ride in the database and a binary column will be used to indicate whether the user is a passenger or driver for that particular ride since drivers are able to be drivers and passengers in the application.

Should a driver be banned from the software, data will be added to the Ban relation. The driver's unique identifier, time of ban, duration of ban, and whether the ban is permanent will be stored in the Ban table.

d.) Access Control and Security¶

RideFind will not require a driver's licence number from a registered driver. Therefore, information encryption will not be used in the program.
The user account information will be stored in the database and accessed only upon login to verify the user.

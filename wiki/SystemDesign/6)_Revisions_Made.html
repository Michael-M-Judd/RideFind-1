6) Revisions Made¶

Subsystem Decomposition¶

As programming began, changes needed to be made to the methods and attributes in the existing class diagram found in Section 3a. The Driver class was removed because it was redundant and was mostly handled by the User class. The db Class was also removed because the only methods it contained were Connect and Disconnect so the group felt that these classes could be handled by the dbHelper class.

The parameters passed into the dbHelper class were changed due to circular dependency errors. The circular dependency errors were occurring because User and Ride were both referencing dbHelper to call the database but dbHelper was also calling User and Ride to use these classes. To resolve this error, parameters from Ride and User were handled in their respective classes and they were passed in the correct format to the dbHelper class. This meant that dbHelper no longer needed to reference User and Ride, therefore removing the circular dependency.

Since the GUI was programmed using C#, another change that needed to be made was changing unmanaged classes to managed. Then, it was noticed that the managed classes don't support string (std::string). Therefore, we had to use String^ (System::String).
